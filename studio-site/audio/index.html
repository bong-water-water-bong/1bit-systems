halo-audio is the voice lane of the halo-ai stack — whisper STT on the way in, halo-server 1-bit inference in the middle, kokoro TTS on the way out. All three share the same 128 GB LPDDR5 memory pool on a single Strix Halo APU. No PCIe tax, no cloud round-trip, no subscription gate. The business case is simple: your voice never leaves the machine.

the stack

halo-whisper · ear

ggerganov/whisper.cpp on gfx1151. Single-pass today; sliding-window streaming with 500 ms partials on the roadmap. plan →

halo-server · brain

Native HIP ternary GEMV. OpenAI-compat /v1 SSE stream. Feeds token deltas to halo-voice as they arrive — not after the reply closes.

halo-kokoro · mouth

olokobayusuf/kokoro.cpp fork (stampby/halo-kokoro). Single-header C++, onnxruntime. 24 kHz mono WAV per sentence.

halo-voice · conductor

Rust sentence-boundary splitter. LLM SSE → splitter → parallel TTS POSTs → WAV chunks streamed to any stdout sink (paplay, ffplay, your app).

halo-echo · wire

axum WebSocket server. Wraps halo-voice, encodes 20 ms Opus frames with audiopus + libopus, preamble-then-packets shape. Browser + mobile friendly.

halo-mcp · hands

stdio MCP bridge. 19 tools: 17 specialists + skill_manage + memory_manage. Claude Code, Hermes, or any MCP client can drive the voice loop plus mutate the agent's on-disk skills + memory files.

why this is a business

voice data never leaves

Medical dictation, legal transcription, private journaling, kids' devices — any audio use case where cloud STT is a non-starter. Strix Halo + halo-audio is a drop-in compliant surface.

cost structure

One mini-PC. No per-minute STT billing, no per-character TTS billing, no per-token inference billing. Pays for itself in a month against a small clinic's whisper-api line item.

latency floor

Round-trip-to-cloud is 150-400 ms before any compute starts. On-APU is 0 ms. For real-time voice agents, that's the difference between "conversational" and "walkie-talkie."
