Benchmarks

Pinned numbers for halo-1bit-2b on the production box (strixhalo, Ryzen AI MAX+ 395 / gfx1151, 128 GB LPDDR5), greedy sampling, 2026-04-23. Full methodology, curves, rocprof breakdowns, cross-arch parity, and reproduction steps on the wiki.

Top line

Kernel-direct bitnet_decode:   80.8 tok/s   (ctx 64, 3-round mean)
Kernel-direct @ ctx 1024:      68.2 tok/s
HTTP 1bit-server wall-clock:   65.7 tok/s   (inline sampler, int8 KV default)
Prefill:                       88.3 tok/s

PPL wikitext-103 chunk-1024:    9.16
PPL wikitext-103 pass-4095:    11.98

The HTTP figure is wall-clock (prefill + axum + JSON serialize). The kernel figure is decode-only.

Kernel breakdown

  • Ternary GEMV at 92 % of LPDDR5 peak per rocprof bytes-read — bandwidth-bound.
  • Split-KV Flash-Decoding attention up to 11.98× vs single-block at L=8192, bit-exact (default everywhere).
  • Activation sparsity 79.91 % (above Nurvitadhi's 50–60 % prediction; upper bound on speedup ~10–15 % since decode is weight-bound).
  • WMMA issue-rate probe: 50.17 TFLOPS fp16 / 53.60 TOPS int8 (42 % of theoretical 118.8 — realistic matmul ceiling).

Cross-arch parity

Same .h1b, same kernel sources, run on Strix Halo (gfx1151) and RX 9070 XT (gfx1201). Output is bit-exact. delta_ppl = 0.000000. Perf per watt: strixhalo wins 2.06× (0.904 tok/s per W vs 0.439); gfx1201 wins absolute tok/s by 1.17× at 176 W vs 73 W.

Stability

OPTC Tier-3 soak (amdgpu.dcdebugmask=0x410), 2 hours concurrent bitnet_decode + sd-cli SDXL: 703 samples, 0 new OPTC events, util 100 %, edge max 65 °C, 0 REG_WAIT lines in dmesg-post.

What makes a bench honest

  1. One hardware, one binary, one model. Cross-arch comparisons are labeled.
  2. Real-backend only. Stub-backend numbers do not count.
  3. Multiple rounds. A single run is an anecdote; three rounds with variance reported is a benchmark.
  4. Source-of-truth JSON. Every row links to the file it came from.
  5. Denominator labeled. Decode-only, prefill-included, and wall-clock-with-HTTP are all valid; each row says which.
