FAQ

Six questions asked often enough in Discord that they deserve a link. Longer list on the wiki.

1. Does this work on my RX 7900 / 9070 / 4090?

No. Today's kernels are written for gfx1151 specifically. Other AMD cards might compile but LDS tiling is tuned for Strix Halo's L1 — you'll probably get the code running and then be very disappointed by the numbers. NVIDIA needs a CUDA port we haven't written. We'll expand hardware support once the Strix Halo path is solid and shipping.

2. Do I need CachyOS, or will Ubuntu / Arch / Fedora work?

We develop on CachyOS 7.x because it's what the target owner runs. Arch should work identically (CachyOS is Arch-derived). Ubuntu + Fedora might — the install script does Arch/pacman things. Ports are welcome; we just won't own the bug.

3. Is my data leaving the box?

No. The installer doesn't phone home. 1bit-server runs on localhost by default; mesh access is opt-in via Headscale. There are no analytics pixels on this website. We don't ship a telemetry agent.

4. Can I use it with Claude Code / Open WebUI / Hermes?

Yes — anything OpenAI-compat works. Point the base URL at https://<your-box>/v1 and the bearer from /etc/caddy/Caddyfile. Open WebUI is the one we test against most.

5. How does the NPU story actually look?

Honestly: the XDNA 2 tile is there, Lemonade 10.2 shipped Linux NPU 2026-04-20, and the primary lane for us is ONNX Runtime + VitisAI EP (AMD's official path). The VitisAI EP shim for Strix Halo is currently login-gated inside the RyzenAI 1.7.1 bundle; we're working on extracting it. Ballpark: prefill offload for prompts ≥ 33 tokens. Not shipping today. See the wiki page Why no NPU yet.

6. Why is it called 1bit when the weights are 1.58 bits?

"1.58 systems" reads badly. The class of models is widely called "1-bit LLMs" (Microsoft's phrasing). We match the common name for the model class. If this offends your information theory, we sympathize — log (3)/log (2) ≈ 1.585 and we know.
