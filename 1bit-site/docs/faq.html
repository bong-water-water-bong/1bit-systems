FAQ

Ten questions asked often enough in Discord that they deserve a link.

1. Does this work on my RX 7900 / 9070 / 4090?

No. Today's kernels are written for gfx1151 specifically. Other AMD cards might compile but LDS tiling is tuned for Strix Halo's L1 — you'll probably get the code running and then be very disappointed by the numbers. NVIDIA needs a CUDA port we haven't written. We'll expand hardware support once the Strix Halo path is solid and shipping.

2. Do I need CachyOS, or will Ubuntu / Arch / Fedora work?

We develop on CachyOS 7.x because it's what the target owner runs. Arch should work identically (CachyOS is Arch-derived). Ubuntu + Fedora might — the install script does Arch/pacman things. Ports are welcome; we just won't own the bug.

3. How much RAM do I need?

128 GB LPDDR5 is the Strix Halo configuration we target. 32 GB and 64 GB SKUs will run the stack but you'll feel it at long context. KV cache at L=2048 is ~320 MB per session.

4. Is my data leaving the box?

No. The installer doesn't phone home. 1bit-server runs on localhost by default; mesh access is opt-in via Headscale. There are no analytics pixels on this website. We don't ship a telemetry agent.

5. Can I use it with Claude Code / Open WebUI / Hermes?

Yes — anything OpenAI-compat works. Point the base URL at https://<your-box>/v1 and the bearer from /etc/caddy/Caddyfile. Open WebUI is the one we test against most. Hermes integration is tracked in Hermes integration.

6. How does the NPU story actually look?

Honestly: the XDNA 2 tile is there, Lemonade 10.2 shipped Linux NPU 2026-04-20, and the primary lane for us is ONNX Runtime + VitisAI EP (AMD's official path). The VitisAI EP shim for Strix Halo is currently login-gated inside the RyzenAI 1.7.1 bundle; we're working on extracting it. Ballpark: prefill offload for prompts ≥ 33 tokens. Not shipping today. See Why no NPU yet.

7. What's the long context story?

fd-attn (split-KV Flash-Decoding) holds the tok/s curve reasonably well out to L=4096. Beyond that we rely on the rope-tuning fix from 2026-04-19 that drops PPL from ~520 to ~12 on wikitext — but it's not magic. Context past L=8192 needs more work.

8. Do I need to be online to install?

The first install needs internet for pacman deps + model downloads. After that, 1bit-server runs fully offline. You can rsync the model cache from another box and skip the download step — see Path 2.

9. Is this a fork of bitnet.cpp?

No. We run Microsoft's BitNet b1.58 2B-4T checkpoint, but the inference stack (HIP kernels, Rust orchestrator, HTTP server, voice lane, everything above) is ours. Microsoft's bitnet.cpp is CPU + CUDA; we're gfx1151 first with CPU and NPU lanes joining the aggregator.

10. Why is it called 1bit when the weights are 1.58 bits?

"1.58 systems" reads badly. The class of models is widely called "1-bit LLMs" (Microsoft's phrasing). We match the common name for the model class. If this offends your information theory, we sympathize — log(3)/log(2) ≈ 1.585 and we know.
