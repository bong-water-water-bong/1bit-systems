Desktop (1bit-helm)

Native Rust (egui/eframe). No Electron, no CEF, no web view. Ships with the installer.

Scope status. Helm is expanding from thin-client to full embedded inference engine. Not all panes are wired yet — the chat + status panes are the anchors; model manager, power controls, voice pane are in progress. See Crate-halo-helm.

Launch

1bit-helm         # from a terminal
# or:  click the 1bit systems entry in your desktop launcher

Panes

  • Chat — model picker, system-prompt slot, stream toggle, token meter.
  • Model manager — list local, browse HF, pull + convert via gguf-to-h1b / bitnet-to-tq2.
  • Power — CPU / iGPU / NPU toggles; wraps FlyGoat/RyzenAdj on Linux.
  • Status — tok/s, memory, thermals, per-lane dispatch counters. Pulls from /metrics.
  • Voice — mic button, Whisper + Kokoro flow. Same loop as the PWA, just local.
  • Settings — bearer rotation, mesh node access, log levels.

Two run modes

  • Default (service mode). Helm talks to strix-server over localhost:8180. Same state across multiple clients; mesh access works; headless if you close helm.
  • Opt-in single-process. Helm embeds Arc<Router> in-process, skips HTTP. Lowest overhead — laptop mode. Enable via --embedded.

Theme

Dark-first, cyan accent matching the website. CachyOS theme integration picks up accent colour if your desktop has one.

Out of scope (deliberately)

  • Electron / Tauri / CEF — Rule A adjacent, killed by choice. Egui stays native.
  • Plugin ecosystem. Helm is the first-party surface; plugins add complexity before we know what people want.
  • Mobile. The PWA at /voice/ is the phone story.
