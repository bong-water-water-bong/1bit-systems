Models

We serve ternary models (1.58-bit weights in {-1, 0, +1}). Stock catalog + how to pack your own.

Stock catalog

All three ship by default. Measured 2026-04-20 on the strixhalo box — wikitext-103, burn-in-harness.

Formats

  • TQ1 — 1.58 bits/weight, 5 ternaries packed per byte. Microsoft's bitnet.cpp layout. Our native first-class citizen.
  • TQ2 — 2.0 bits/weight (sign + magnitude). Slightly looser, slightly faster to dequant on the iGPU path. Bonsai upstream uses this.
  • H1B — our container. Memory-mapped first; header describes the ternary pack and tokenizer. See Why .h1b.

Convert a GGUF

Already packed as TQ1 or TQ2 in a GGUF? One command:

gguf-to-h1b \
  --input model-TQ1.gguf \
  --output ~/.cache/1bit systems/models/myname.h1b

The tool is a Rust CLI in tools/gguf-to-h1b/. Validates the pack, extracts the tokenizer, writes a header. 3–10 seconds for a 2B model.

Requantize a fp16 BitNet checkpoint

Microsoft-style BitNet fp16 checkpoint that hasn't been packed yet? Run the requantizer:

bitnet-to-tq2 \
  --input path/to/bitnet-fp16/ \
  --output ~/.cache/1bit systems/models/mybitnet.h1b \
  --activation hadamard-a4        # optional, for BitNet v2 recipe

This calls through to Python in tools/bitnet-to-tq2/ — Python is allowed here because it runs once on a dev box and exits. See Rule A.

List & pick

1bit list models
1bit set default halo-v2
1bit bench halo-v2       # kicks a tok/s + PPL run

Budget

A 2B ternary model at TQ1 weighs ~430 MB on disk. KV-cache at L=2048, fp8 K + fp16 V, ~320 MB per active session. 128 GB LPDDR5 holds a lot of concurrent sessions. Memory is free here — bandwidth is the ceiling.
