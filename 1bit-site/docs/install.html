Install

Three paths, one end state: a running strix-server.service, a stocked model cache, and a Caddy bearer on :8180. AppImage is the one most people use.

Support matrix. Officially: AMD Ryzen AI MAX+ 395 (Strix Halo, gfx1151) on CachyOS 7.x. Other distros might work; we won't fix bugs against them until someone confirms. See Architecture § targets.

AppImage

curl -fsSL https://github.com/bong-water-water-bong/1bit-systems/releases/latest/download/1bit-systems-x86_64.AppImage -o 1bit-systems.AppImage
chmod +x 1bit-systems.AppImage
./1bit-systems.AppImage server

Single self-contained binary: HIP kernels, Rust runtime, tokenizer, the lot. First run seeds ~/.cache/1bit systems/models/ and prints a bearer token. No root needed unless you want to wire the bundled systemd unit.

AUR

yay -S 1bit-systems       # or: paru / pikaur
systemctl --user enable --now strix-server

The PKGBUILD lives under strixhalo/pkgbuild/ and installs binaries into /usr/bin/, user-systemd units into /usr/lib/systemd/user/, and a default Caddyfile into /etc/caddy/. Pulls a pre-built rocm-cpp object for the current kernel — faster than compiling, but you don't get machine-specific tuning. Rebuild from source if you want that.

Source

git clone https://github.com/bong-water-water-bong/1bit-systems ~/1bit-systems
cd ~/1bit-systems
./install.sh

What it does, in order:

  • Detects the SoC (lscpu | grep -i "AMD Ryzen AI MAX+"). Aborts on a mismatch with a clear message.
  • Installs pacman deps from packages.toml. Runs unprivileged; prompts for sudo only on pacman.
  • Builds HIP kernels in rocm-cpp/. 45 min to 4 h depending on cache warmth; progress bar is honest.
  • Runs cargo build --release --workspace. Installs binaries into ~/.local/bin/.
  • Enables user-systemd units (systemctl --user enable --now strix-server, etc.).
  • Prints the bearer, the URL, and a curl test line.

Idempotent. Rerun anytime to repair or upgrade.

Verify

systemctl --user status strix-server
curl -s http://localhost:8180/health
1bit status

Expected: the service active, /health returns ok, and 1bit status prints a summary of running units + backend mode.

Uninstall

1bit doctor --purge
# or manually:
systemctl --user disable --now strix-server strix-whisper strix-kokoro strix-echo
rm -rf ~/.local/share/1bit-systems ~/.local/bin/1bit-* ~/.config/systemd/user/strix-*.service

Config files in ~/.config/1bit systems/ and the model cache in ~/.cache/1bit systems/ are left alone. Remove them by hand if you want a clean slate.
