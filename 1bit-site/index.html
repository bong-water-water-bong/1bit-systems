Four pillars

A small stack. Each piece does one thing and does it on bare metal.

Ternary at native speed

1.58 bpw weights, hand-written HIP kernels on gfx1151. Ternary GEMV runs at 92% of LPDDR5x peak. No hipBLAS in the runtime path.

NPU dispatch

XDNA 2 (AIE2P) toolchain proven end-to-end. IRON axpy 160/160, matmul 512×512×512 i8 bit-exact. Custom Peano kernels via libxrt.

OpenAI · Ollama · Anthropic compatible

Drop-in replacement on :8180. /v1, /api/v1, /v0, /api/v0, Ollama /api/*, Anthropic /api/messages, WS realtime.

Bare metal first

Single ELF, systemd, no Docker, no interpreter. Engine in-process, weights mmap’d. Weights and prompts never leave the machine.

What runs out of the box

Components shipped in the AppImage. Each is a single binary or a single unit.

lemond

Local LLM gateway on :8180. C++. Recipe-dispatched to the in-process Engine and external backends.

1bit-flow

DAG workflow engine. Twelve in-tree ctest cases. The plumbing for agent loops and pipelines.

bun-fight

Eight-LLM debate harness. Three ctest cases. Adversarial round-tabling for hard prompts.

1bit-rag

Hybrid retrieval — dense + lexical. Eight ctest cases. Embeddings on CPU, rerank on iGPU.

agent-core

GAIA-derived agent spine. Tool-use, planner, and memory loop, ported into the C++ core.

halo-arcade

Eight-game retro cabinet. Vanilla JS canvas, no framework. Lives at 1bit.gg.

1bit-helm

Desktop client. Rust egui/eframe. SNI tray icon for service start/stop on Plasma.

1bit-cli

Operator commands. 1bit status, logs, doctor, install, update.

Modalities

Hardware

Built first for AMD Strix Halo. Same fat binary covers eight consumer AMD Wave32-WMMA arches.

gfx1151 iGPU

  • RDNA 3.5, ~RTX 4070 class
  • 256 GB/s LPDDR5x, 128 GB unified
  • Wave32 WMMA, native FP16 / BF16
  • Hand-written HIP, no hipBLAS

XDNA 2 NPU

  • 50 TOPS, AIE2P tile array
  • MLIR-AIE → Peano → libxrt
  • i8 matmul + axpy bit-exact today
  • Ternary GEMV kernel in flight

Also runs on gfx1201 (RX 9070 XT, RDNA 4) and gfx1100 (RDNA 3). One fat binary, eight arches.

Architecture

One ELF per service, single Caddy front, no Docker. Browser, CLI, and mobile all hit the same gateway.

Each service is a systemctl --user unit. Stop one, the rest keep running. 1bit doctor probes every endpoint.

Install

Three paths. Pick one. Arch / CachyOS is our truth; the AppImage covers everything else.

yay -S 1bit-systems-bin

AUR. Pulls the AppImage, drops a launcher in ~/.local/bin, registers the systemd user units.

curl -fsSLO https://github.com/bong-water-water-bong/1bit-systems/releases/download/v0.1.0/1bit-systems-0.1.0-x86_64.AppImage
echo "8a964f89bdef68ed914c04fcf23092ac642c424bb70f74dc10e8558e93b94036  1bit-systems-0.1.0-x86_64.AppImage" | sha256sum -c -
chmod +x 1bit-systems-0.1.0-x86_64.AppImage
ln -sfn "$PWD/1bit-systems-0.1.0-x86_64.AppImage" ~/.local/bin/1bit

Requires glibc ≥ 2.35 and Linux kernel 6.x. ROCm not bundled — install on the host first.

curl -fsSL https://1bit.systems/install.sh | sh

Bootstraps ROCm against gfx1151, builds the workspace, installs the user units. Read it first; it’s 400 lines.

GAIA UI & mobile pairing

A vendored copy of amd/gaia ships in the AppImage. Same React UI AMD ships, but the FastAPI backend is replaced by a thin C++ shim that calls our gateway. Open http://localhost:8000/gaia/ on the host or pair a phone over the mesh.

What runs at /gaia/

Vendored React UI from amd/gaia served as static assets by Caddy. The FastAPI surface is reimplemented in C++ and proxied through to lemond :8180: completions, embeddings, model-list, voice loop. Zero Python in the runtime path.

WebSocket realtime fans out to 1bit-voice for sentence-boundary TTS streaming. Whisper partials drive the prompt input.

Source: 1bit-services/gaia-shim/ · upstream React tree pinned at amd/gaia@v0.9.x.

Mobile pairing flow

1.Helm generates a Headscale preauthkey, encodes the host URL + key as a QR.
2.Phone scans QR with Tailscale (custom login server = your Headscale).
3.Phone joins the mesh as a leaf node, gets a 100.64.x.y address.
4.Browse http://halo.tail-NNN.ts.net:8000/gaia/ — chat, voice, image gen.
5.Bearer token auto-injected by Caddy on the mesh interface only.

No data leaves the mesh. Caddy refuses non-mesh sources outside localhost.

Why this matters

The pitch in four cards.

1.58 bpw weights

Ternary quantization stores each weight in a sign + zero-pos pattern. ~10× smaller than FP16, ~4× smaller than Q4. Whole 2 B model in 410 MB.

Memory-bound decode

LLM decode is bandwidth-bound, not FLOP-bound. Strix Halo's 256 GB/s LPDDR5x is the budget; we use 92% of it. Smaller weights = more tokens per second of bandwidth.

$2 000 mini-PC

One Strix Halo box runs the whole stack. No GPU farm, no cloud bill, no API key. Compare $0.50–$5 per million tokens cloud vs. ~$0 per token at the wall after install.

No Python at runtime

Single ELF, systemd unit, weights mmap'd. No interpreter to start, no GIL to fight, no requirements.txt to drift. Cold boot to first token ~30 s. Reboot survival = clean.
