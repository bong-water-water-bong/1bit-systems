v0.2 · Strix Halo · gfx1151 + XDNA 2

1bit.systems — the 2-bit killer. 1-bit is the floor. Sub-2-bit MoE is what you'll keep open all day.

Built on upstream Lemonade Server and FastFlowLM. OpenAI-compatible endpoint on :13305. Bonsai-1.7B at IQ1_S decodes ~280 tok/s as the floor; Qwen3.5-35B-A3B at IQ2_XXS holds ~73 tok/s at half the disk of the Q4_K_XL baseline. Bring your own client.

Install GitHub

What this is

One install script

Bash. Installs lemonade-server, fastflowlm, xrt, amdxdna, ROCm. Writes memlock limits. Pulls a default 1-bit model. Idempotent.

One control CLI

1bit up, 1bit status, 1bit pull, 1bit bench. ~120 lines of bash. That's the whole product surface.

One OpenAI endpoint

http://127.0.0.1:13305/v1/*. Works with any client that speaks OpenAI — AMD GAIA, Open WebUI, Continue, raw curl. We don't ship UI.

Three lanes, one shape

ROCm + Vulkan llama.cpp on the iGPU; FastFlowLM on the XDNA 2 NPU. All three speak OpenAI. Pick the lane per workload.

Three lanes

Each lane is a swap-in inference backend behind the same wire shape.

GPU · ROCm

llama.cpp HIP, prebuilt for gfx1151 by Lemonade. Strong prefill. Quants: Q4_K_M, Q2_K, IQ1_S, TQ1_0, TQ2_0.

GPU · Vulkan

llama.cpp Vulkan. Best decode on sub-2-bit quants today — mainline IQ1_S compute shaders win on Strix Halo.

NPU · XDNA 2

FastFlowLM. UINT4-AWQ / q4nx models pre-compiled by AMD. Low-power inference, runs in parallel with the iGPU.

Install

CachyOS / Arch on Strix Halo. One script. Idempotent.

Step 0 — prerequisites

A clean CachyOS / Arch base ships most of these, but a minimal install can be missing some. Run once before cloning:

sudo pacman -Syu                                  # fully update first
sudo pacman -S --needed git base-devel paru curl  # git for clone, base-devel for AUR builds,
                                                  # paru for the AUR lemonade-server package,
                                                  # curl for the model pull

You'll also need:

  • A user with sudo privileges — the installer writes /etc/security/limits.d/99-1bit-systems.conf and patches Lemonade's manifest under /usr/share/.
  • A Strix Halo APU (gfx1151 iGPU + XDNA 2 NPU). Other Ryzen AI gens (XDNA 1 — 7000 / 8000 / 200-series) will install but the flm:npu lane won't light up.
  • A modern kernel with in-tree amdxdna — Linux 7.0+ ships it. CachyOS 7.0.x and Arch's mainline linux are good. For older kernels (6.18 / 6.19) install paru -S amdxdna-dkms first.
  • ~70 GB free on /home — covers the default 1-bit Bonsai pull and headroom for the 35B IQ2_XXS daily driver alongside it.

Step 1 — clone + run

git clone https://github.com/bong-water-water-bong/1bit-systems
cd 1bit-systems
./install.sh                 # actually install
./install.sh --dry-run       # preview every action without mutating

Installs lemonade-server, fastflowlm, xrt, xrt-plugin-amdxdna, rocm-hip-sdk. Writes memlock limits. Auto-patches Lemonade's flm:npu version pin to match your installed FastFlowLM (the silent update_required gotcha). Pulls the default 1-bit model (Bonsai-1.7B-IQ1_S, 385 MB). Installs the 1bit CLI to /usr/local/bin/.

After first install: re-login or reboot once so the memlock limits apply for the NPU lane.

Install paths — what AMD says vs what actually works

AMD's docs say "Ubuntu 24.04 only." That's accurate for one of two AMD-built stacks on Strix Halo. We tested all three install routes; they don't give you the same thing. Pick the one that matches what you need.

The two stacks

StackRunsLicenseDistribution
Ryzen AI 1.7.1 EP (closed)AMD's 200+ AWQ/OGA HF checkpoints (huggingface.co/amd/*_rai_1.7.1_npu_*)Closed.deb, Ubuntu 24.04 only, behind account.amd.com
Lemonade Server + FastFlowLM (open)FLM's curated catalog (qwen3, gemma3, phi-4-mini, lfm2, llama-3.2, deepseek-r1, gpt-oss)MIT + ApacheAUR (Arch / CachyOS), installer (Ubuntu), Linux build everywhere

The closed EP runs models the open stack can't. The open stack runs on distros the closed EP can't. Neither is a superset.

Three install paths

Path A — Ubuntu 24.04 native (both stacks)

AMD's .deb bundles install cleanly. The xrt_plugin-amdxdna 2.21.260102.53 kernel module DKMS-builds against Ubuntu's stock kernel. Closed EP runs the AWQ/OGA models. Lemonade + FLM also works alongside.

Trade-off: you have to run Ubuntu.

→ Detailed step-by-step runbook (10 steps) on GitHub

Path B — Distrobox on a non-Ubuntu host

Looks tempting. Mostly doesn't work for the closed EP. Tested on CachyOS host (kernel 7.0.2-1-cachyos) → Ubuntu 24.04 distrobox container with NPU passthrough:

  • ✅ xrt-smi examine sees the NPU through /dev/accel/accel0
  • ✅ AMD's quicktest.py (basic ONNX via VitisAIExecutionProvider) passes
  • ❌ onnxruntime-genai-ryzenai LLM init fails with Generic Failure

Cause: the strict RyzenAI provider checks something the AMD-shipped kernel module exposes that the in-tree amdxdna doesn't. Distrobox passes the device through; it cannot give the container its own kernel module.

Trade-off: distrobox gets you 80% of the way and falls down at the actual LLM workload. If you want the closed EP, you're booting Ubuntu 24.04 native.

→ Detailed runbook + failure-mode trace on GitHub

Path C — Native non-Ubuntu (Arch / CachyOS / Fedora / …)

Lemonade + FastFlowLM works directly on the host. No container, no DKMS for closed-EP modules. The in-tree amdxdna (kernel 7+) or xrt-amdxdna DKMS (kernel 6.x) is enough. This is what this repo installs (see Install above). All four lanes light up:

LaneBackendRecipe
NPU (XDNA 2)FastFlowLMflm:npu
iGPU (gfx1151) — Vulkanllama.cppllamacpp:vulkan
iGPU (gfx1151) — ROCmllama.cppllamacpp:rocm
CPU (Zen 5)llama.cppllamacpp:cpu

Trade-off: no closed EP. AMD's 200+ AWQ/OGA checkpoints are downloadable but unrunnable on the NPU. FLM's catalog overlaps on the popular architectures.

iGPU is faster than NPU on these sizes. The NPU lane's value isn't peak tg — it's offloading the iGPU for ROCm bigger-model serving, lower power for background inference, and a unified :13305 API where the request picks the silicon per model name.

What's still missing on Linux

  • AWQ/OGA on non-Ubuntu — needs an xrt-based EP equivalent of the closed .so. Doesn't exist publicly.
  • Brevitas in optimum-amd — “Coming soon” since 2024.
  • Diffusion + CLIP on NPU under Linux — Windows-only.
  • BitNet on NPU — research-tier; not shipped on any consumer NPU yet.

60-second 1-bit

Speed-to-running is the claim to fame. Pulled the model, asked it to write a haiku, got an answer, end-to-end through Lemonade's OpenAI endpoint on :13305. Measured 2026-04-28 on the production strixhalo box (Ryzen AI MAX+ 395 / gfx1151, 124 GB LPDDR5x).

# 1. one-time pull (385 MB at ~85 MB/s on the box ≈ 4-5 s)
lemonade pull lilyanatia/Bonsai-1.7B-requantized:Bonsai-1.7B-IQ1_S.gguf

# 2. ask it for a haiku
curl -s http://127.0.0.1:13305/v1/chat/completions \
  -H 'Content-Type: application/json' \
  -d '{"model":"user.Bonsai-1.7B-requantized-Bonsai-1.7B-IQ1_S.gguf",
       "messages":[{"role":"user","content":"Write a haiku about local AI inference on a Strix Halo APU."}],
       "max_tokens":80}'

Output (live, 1-bit Bonsai-1.7B at IQ1_S):

In a world of code, I see
Strix's power in the shadows,
Inference's rhythm, a song.

Numbers:

  • Decode: 284 tok/s (27 tokens / 95 ms)
  • Prefill: 119 tok/s (28 tokens)
  • Cold wall (model load + chat): 3.9 s
  • Model on disk: 385 MB

1.7B params at sub-2-bit precision on an integrated GPU, behind the same OpenAI API you'd use for any other model. Upstream llamacpp:rocm on Lemonade.

The floor and the daily driver

1-bit Bonsai-1.7B is the floor — raw decode, the receipt that sub-2-bit on this hardware is real. Most users won't run a 1.7B as their assistant. The daily driver is sub-2-bit on a real-sized model. Same hardware, same Lemonade endpoint, captured 2026-04-28 head-to-head:

ModelQuantBits/wtDiskDecode (med, 200 tok)Decode (long, 350 tok)
Qwen3.5-35B-A3B (MoE) baselineQ4_K_XL~4.522 GB54 tok/s54 tok/s
Qwen3.5-35B-A3B (MoE) sub-2-bitIQ2_XXS~2.011.5 GB73 tok/s73 tok/s
Δ sub-2-bit / baseline0.52×+35%+35%

Half the disk, ~35% faster decode, on a real 35B-class MoE that actually answers real questions. Sub-2-bit on MoE is the sweet spot on this APU. 1-bit is the headline (Bonsai above); 2-bit is what you'll keep open all day. lemonade pull Manojb/Qwen3.5-35B-A3B-UD-IQ2_XXS.gguf + lemonade load Qwen3.5-35B-A3B-UD-IQ2_XXS.gguf-UD-IQ2_XXS and you're there.

Quickstart

# starts lemond on :13305
# opens the lemonade webapp in your browser
# launches AMD GAIA if it's installed

1bit status
# ✓ lemond: up at http://127.0.0.1:13305
# ✓ flm:    serving
# memlock limit: unlimited

1bit pull qwen3:0.6b      # auto-routes to NPU (flm) or GPU (lemonade)
1bit bench                # full sub-2-bit / ternary sweep
1bit down                 # stop lemond

OpenAI API

Standard OpenAI shape on http://127.0.0.1:13305/v1/*. No auth required for localhost.

curl -s http://127.0.0.1:13305/v1/chat/completions \
  -H "content-type: application/json" \
  -d '{
    "model": "Bonsai-1.7B-IQ1_S",
    "messages": [{"role": "user", "content": "hello"}],
    "stream": false
  }'

Endpoints: /v1/chat/completions (streaming + non-streaming), /v1/completions, /v1/embeddings, /v1/models. NPU lane runs on a separate port (see flm port) with the same shape.

Compatible apps

Anything that speaks OpenAI works. We don't ship UI — pick yours.

Logo provenance: /assets/apps/NOTICE

AMD GAIA

amd-gaia.ai · Electron desktop, MIT, .deb on Linux. Point its Lemonade endpoint at http://127.0.0.1:13305/v1/. 1bit up launches GAIA automatically if it's on $PATH.

Open WebUI

Browser chat UI. Set the OpenAI-compatible base URL to http://127.0.0.1:13305/v1. Works out of the box.

Continue (VS Code / JetBrains)

IDE assistant. Add a custom OpenAI provider with the same base URL. Code completion + chat.

Anything else

OpenAI Python / TS SDKs, LangChain, LlamaIndex, Aider, Claude Code, raw curl. If it talks OpenAI, it talks to us.
