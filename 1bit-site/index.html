What this actually is

After the cutover. Boring on purpose.

One install script

Bash. Installs lemonade-server, fastflowlm, xrt, amdxdna, ROCm. Writes memlock limits. Pulls a default 1-bit model. Idempotent.

One control CLI

1bit up, 1bit status, 1bit pull, 1bit bench. ~120 lines of bash. That's the whole product surface.

One OpenAI endpoint

http://127.0.0.1:13305/v1/*. Works with any client that speaks OpenAI — AMD GAIA, Open WebUI, Continue, raw curl. We don't ship UI.

Three lanes, one shape

ROCm + Vulkan llama.cpp on the iGPU; FastFlowLM on the XDNA 2 NPU. All three speak OpenAI. Pick the lane per workload.

Three lanes

Each lane is a swap-in inference backend behind the same wire shape.

GPU · ROCm

llama.cpp HIP, prebuilt for gfx1151 by Lemonade. Strong prefill. Quants: Q4_K_M, Q2_K, IQ1_S, TQ1_0, TQ2_0.

GPU · Vulkan

llama.cpp Vulkan. Best decode on sub-2-bit quants today — mainline IQ1_S compute shaders win on Strix Halo.

NPU · XDNA 2

FastFlowLM. UINT4-AWQ / q4nx models pre-compiled by AMD. Low-power inference, runs in parallel with the iGPU.

Install

CachyOS / Arch on Strix Halo. One script. Idempotent.

git clone https://github.com/bong-water-water-bong/1bit-systems
cd 1bit-systems
./install.sh

Installs pacman + AUR packages, writes /etc/security/limits.d/99-1bit-systems.conf (memlock unlimited), pulls the default 1-bit model (Bonsai-1.7B-IQ1_S, 385 MB), installs the 1bit CLI to /usr/local/bin/.

After install: re-login or reboot once so the memlock limits apply for the NPU lane.

  • AUR package 1bit-systems-bin. → not shipped — just the install script.
  • AppImage covers everything else. → not shipped.
  • Source build via cmake --preset release-strix. → no source build — nothing in this repo to compile.

OpenAI API

Standard OpenAI shape on http://127.0.0.1:13305/v1/*. No auth required for localhost.

curl -s http://127.0.0.1:13305/v1/chat/completions \
  -H "content-type: application/json" \
  -d '{
    "model": "Bonsai-1.7B-IQ1_S",
    "messages": [{"role": "user", "content": "hello"}],
    "stream": false
  }'

Endpoints: /v1/chat/completions (streaming + non-streaming), /v1/completions, /v1/embeddings, /v1/models. NPU lane runs on a separate port (see flm port) with the same shape.

Cutover note

This project pivoted on 2026-04-27 from a 7-lane C++23 tower with custom HIP and Peano kernels to the lean install + control plane you see today. The previous tree (cpp/, rocm-cpp/, npu-kernels/, agents/, browser/, strixhalo/, packaging/) is preserved on the archive/cpp-tower-2026-04-27 branch — nothing was lost — but it does not build against current upstream toolchains and is no longer the path forward.

The honest read: the in-house ternary engine is the differentiator → the in-house engine was scope creep. Mainline llama.cpp's Vulkan IQ1_S shaders already deliver 278 tok/s decode on Strix Halo without any of it. FastFlowLM ships pre-compiled NPU models. The win is wiring those two together cleanly, not reimplementing them.

Admitting defeat is part of the product. The strikeouts on this page are not bugs — they're the trail.
