AMD Strix Halo · gfx1151 + XDNA 2 · inference engine first

1bit.systems is a local inference engine for Ryzen AI. Apps connect over OpenAI-compatible APIs. The control plane comes second.

The public shape is simple: apps talk to an inference endpoint. Use Lemonade as the canonical OpenAI-compatible multimodal and OmniRouter server on :13305, FastFlowLM as the XDNA NPU runtime on :52625, and 1bit-proxy on :13306 when a client wants both lanes behind one OpenAI base URL.

Install GitHub

Inference engine

The engine is the product surface: OpenAI-compatible apps send requests to Lemonade, FastFlowLM, or the union endpoint. The single control plane manages those services after the inference path is clear.

Lemonade

Canonical OpenAI-compatible multimodal inference server and OmniRouter source of truth on http://127.0.0.1:13305/api/v1 and /v1.

FastFlowLM

XDNA NPU runtime on http://127.0.0.1:52625/v1, serving FLM chat models and the local embedding side lane.

1bit proxy

Convenience union endpoint on http://127.0.0.1:13306/api/v1 and /v1. Lemonade remains default; FLM receives targeted model families.

Apps

GAIA, Open WebUI, AnythingLLM, Continue, Dify, n8n, and custom SDK clients connect by setting an OpenAI-compatible base URL.

Control plane

1bit, GAIA, Open WebUI, and 1bit-stack.target manage the engine after the endpoint contract is set.

Open WebUI

Secondary browser UI on :3000, pointed at the union endpoint by the systemd unit.

Install

Source install builds and wires the forked Lemonade/FastFlowLM paths instead of relying on stale upstream application packages.

git clone https://github.com/bong-water-water-bong/1bit-systems
cd 1bit-systems
./install.sh

# after first install, re-login or reboot so memlock limits apply
1bit up
1bit status

The installer writes the CLI, systemd units, Open WebUI configuration, memlock limits, and local service defaults. Lemonade and FastFlowLM are built from maintained fork sources under /opt/1bit.

The five rules

The live stack is Lemonade on :13305, FastFlowLM on :52625, and 1bit-proxy on :13306. These rules describe that stack as it runs now.

Rule A

Core serving stays Python-free. Training, notebooks, build-time conversion, caller-side tools, and isolated compatibility UIs are allowed. The proxy, kernels, native runtimes, and model hot paths stay Python-free.

Rule B

C++20 for kernels. HIP code lives in rocm-cpp/; Rust is for layers above the kernel boundary.

Rule C

hipBLAS is banned in the runtime path. Port the kernel to rocm-cpp/ instead.

Rule D

Rust 1.88+, edition 2024. Bumps require a reason.

Rule E

FastFlowLM is the live XDNA serving lane. Custom NPU kernels use IRON at author-time, then MLIR-AIE, Peano, xclbin, and libxrt from C++ at runtime.

Carve-out

Open WebUI is a secondary compatibility UI behind the union endpoint. It does not become the engine.

Architecture

Apps / SDKs
  -> 1bit-proxy :13306/v1 or :13306/api/v1
       -> Lemonade :13305/api/v1    canonical OpenAI + multimodal + OmniRouter
       -> FastFlowLM :52625/v1      XDNA NPU chat + embeddings + opt-in ASR

Open WebUI :3000 -> 1bit-proxy :13306/v1
Control plane    -> 1bit CLI + GAIA + systemd target

Models

Model policy is pragmatic: use 1-bit and ternary GGUF where they win on the iGPU, use sub-2-bit MoE for daily-driver chat, and use FLM's q4nx/AWQ model catalog for the XDNA NPU lane. Embeddings route to FLM when the model name uses the embed-* family.

FAQ

Is :13306 the new Lemonade?

No. Lemonade remains canonical on :13305. The proxy is a client convenience layer.

Is the NPU shipping?

Yes. FastFlowLM runs the XDNA NPU lane on :52625. It is useful for offload and parallelism, not just peak throughput.

Should I expose this to the internet?

No, not directly. These are local developer services. Put authentication, TLS, and explicit routing in front of anything remote.
