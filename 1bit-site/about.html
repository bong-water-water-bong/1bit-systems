1bit.systems · about

Run modern AI on the machine under your desk.

1bit-halo is a native C++ and Rust stack for sub-2-bit language, speech, and image models on AMD Strix Halo. No Python at runtime. No discrete GPU. No cloud dependency.

The thesis

A motivated person should be able to run modern AI on hardware that fits on a desk, costs less than a mid-range discrete GPU, and draws less power than a kettle.

Strix Halo is the first consumer AMD platform where that's credible. 40 RDNA 3.5 compute units. 128 GB unified LPDDR5X at roughly 270 GB/s. Mini-PC form factor. Under 120 W continuous. It doesn't beat a datacenter GPU. It isn't meant to. It's meant to run real models at conversational latency from a silent room.

The trade-off is quantization. A 70B-parameter model doesn't fit in 128 GB at FP16 — it does at 1.58 bits per parameter. Current ternary-quantized models land within a few percent of full precision on hard evals. The stack is built around ternary and sub-2-bit weights from the ground up, not retrofitted around an existing FP16 path.

What it is

  • A serving stack — kernels, HTTP surface, MCP server, speech and image components.
  • A research line — ternary GEMV, split-KV attention, Sparse-BitNet 3:4, BitNet v2, Medusa speculative decoding.
  • A hardware bet — gfx1151 primary, gfx1201 secondary (RX 9070 XT). Unified memory beats PCIe copies for sub-2-bit.
  • A set of constraints — no Python in the serving path, no runtime hipBLAS, no cloud dependency, no telemetry.

The rules

Rule A — no Python at runtime

Hard rule. No Python interpreter in any serving binary. GIL, packaging churn, interpreter startup, supply-chain risk, and license tangles disqualify it from the hot path. Caller-side tooling is any language you want.

Rule B — C++20 for kernels, Rust for orchestration

Default language for a new component is C++20 if it talks to HIP directly, Rust otherwise. Rust gets safety and ownership where correctness matters most; C++ gets HIP intrinsics, wave32 WMMA, and register-level control that the ternary GEMV depends on.

No runtime hipBLAS

Ever. Native Tensile-generated kernels are allowed; runtime hipBLAS calls are banned because its selection heuristic collapses on our skinny ternary GEMV shape.

Origin

The project started as an efficiency exercise. The question was whether a CAD $3,000 Strix Halo mini-PC could serve as a private, always-on AI workstation from a silent closet — no fans screaming, no power bill spike, no cloud. First experiments used MLX via its ROCm backend; that path died almost immediately because MLX ROCm has no ternary mode and BitNet warmup crashed on every run. The pivot to native HIP followed.

The codebase name was halo-ai, inherited from the original project name. The public brand is 1bit.systems, with components named 1bit-halo-*. They refer to the same thing.
