## Rename — internal → `onebit_halo_*`

C++ / Python / Rust lib-name identifiers cannot start with a digit. Split convention:

- **External** (dir, binary, cargo package, repo): `1bit-halo-<name>` — dashes, digit-leading OK.
- **Internal** (C++ namespace, Python module, Rust `lib.name`): `onebit_halo_<name>` — underscores, letter-leading.

Applied:

- 10 C++ files rewritten (4 `.hpp` + 6 `.cpp`) across `1bit-halo-mcp` and `1bit-halo-brain`.
- Include dirs renamed: `include/halo_mcp/` → `include/onebit_halo_mcp/`; same for `halo_brain`.
- All `namespace halo_mcp { }` / `halo_mcp::` / `#include <halo_mcp/…>` rewrites applied.
- `1bit-halo-brain/CMakeLists.txt` install-dir line updated.
- READMEs for both projects rewritten to point at the new include paths.

Intentionally kept:

- Uppercase macros (`HALO_MCP_TIMEOUT_MS`, `HALO_BRAIN_DB`) — out of namespace-rename scope.

## Sparse-BitNet Run 3 autopsy → Run 4 relaunch

### Run 3 death

Launched 2026-04-22T01:40:44Z. Config: 494M parameters, LR 3e-4, warmup 200, 10B-token budget, 3:4 N:M sparsity layered on BitLinear. Trained cleanly for 500 optimizer steps:

```
[opt-step     50] loss=11.0370 lr=7.50e-05 tok/s=49290 tokens=52.4M
[opt-step    500] loss= 5.7668 lr=2.99e-04 tok/s=49540 tokens=524.3M
```

Loss curve textbook. Throughput steady at 49.5k tok/s. Then:

```
[BAIL] mask integrity broken at step 500: no mask_cache on model
```

### Diagnosis — false-positive bail

`Model.__init__` creates `self.mask_cache = {}` (empty dict) and sets `_save_mask_cache = False` on every `BitLinear` / `SparseLinear` child. `enable_mask_monitoring()` must be called to flip the flag and start populating the dict; without it the cache never fills. `verify_nm_mask()` at step 500 reads the empty dict, Python evaluates `bool({}) == False`, and the "no mask_cache on model" branch fires.

The training was fine. The check was wrong.

Compounded by `--save-every 1000` as the default: the bail at step 500 meant zero checkpoints were ever written. 524M tokens of H200 time vanished to a three-line bug.

### Fix — three patches to `pretrain_sparse_bitnet_qwen_0p5b.py`

1. After `model = Model(margs).to(device).to(dtype)` call `model.enable_mask_monitoring()` — guarded by `if not args.smoke and args.verify_mask_every > 0`.
2. After a successful verify, call `model.mask_cache.clear()` so the next verification window reads fresh masks from subsequent forward passes.
3. Change `--save-every` default from `1000` → `100`. First ckpt lands at step 100; future bails still leave a recent recovery point on disk.

Bootstrap `runpod_bootstrap.sh` updated to pass `--save-every 100` explicitly. Both files rsync'd to `/workspace/` on the pod.

### Run 4 launch — live

- `rocminfo` confirms `gfx1201` — marketing name "AMD Radeon RX 9070 XT".
- `rocm-cpp` CMake configure passed with `-DCMAKE_HIP_ARCHITECTURES=gfx1201 -DCMAKE_HIP_COMPILER=/opt/rocm/llvm/bin/clang++` (note: `hipcc` wrapper is rejected by CMake 4.x — clang++ direct).
- Build kicked; reached `ck_gemm.cpp` before hitting the real blocker.

### Real blocker — WMMA intrinsic mismatch

gfx11 WMMA builtin `__builtin_amdgcn_wmma_f32_16x16x16_f16_w32` does not select on gfx1201. RDNA 4 has a separate intrinsic family with `_gfx12` suffix and different operand types (half8 fragments instead of half16; compressed lane layout).

Relevant intrinsics identified (via `strings /opt/rocm/llvm/bin/clang-22`):

```
__builtin_amdgcn_wmma_f32_16x16x16_f16_w32           ← gfx11 only
__builtin_amdgcn_wmma_f32_16x16x16_f16_w32_gfx12     ← gfx12 family
__builtin_amdgcn_wmma_i32_16x16x16_iu8_w32_gfx12
__builtin_amdgcn_wmma_f32_16x16x16_fp8_fp8_w32_gfx12
__builtin_amdgcn_wmma_i32_16x16x32_iu4_w32_gfx12     ← new: sub-byte 4-bit at 32 K
```

Port work is real, not a flag flip:

1. `_gfx12`-suffixed intrinsics require different register layouts.
2. Packed ternary layout may need realignment.
3. Memory-hierarchy retune for GDDR6 vs LPDDR5X (different bandwidth ceiling, different compute / bandwidth ratio).
4. K-outer tile sweep must be redone from scratch on gfx1201.
5. Non-unified memory adds host→device copy path (not free).
